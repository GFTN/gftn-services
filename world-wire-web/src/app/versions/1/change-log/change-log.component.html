What's new in this release.

  • Combined client-api with callback-api to create a single api service

  • Whitelist participants

  • New WW Admin Service: service stores all FItoFI messages

  • Query Payment Status: transaction reports on distribution account

  • Look up payment details

  • Log system status

  • Get balance of issued digital obligations (DO)

  • Changes to issuing a digital obligations (DO) based on the 12-character naming convention

  • Changing the quote service to restrict returns to include only issuers for digital obligations (DO) pairs

  • Added act on Exchange

  • Consume Blacklist or Whitelist data for transaction and quotes

  • Single endpoint for trust operations

  • Refactoring Fees endpoint

  • PaymentListener: Payment listener is added as its own micro service which calls the callback endpoint /payment when receives incoming payment notifications from stellar for a particular participant account

  • Terminate Network Account: WW admin can suspend a Participants User accounts from using the WW network once they have failed to meet the standards outlined or if they request suspension due to a data or security breach of their internal systems
